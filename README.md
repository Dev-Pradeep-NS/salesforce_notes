# Salesforce Developer Notes Starter Pack

This pack is organized around your completed syllabus and is designed for quick revision, interview prep, and practice.

It is now shaped specifically for Salesforce Platform Developer I preparation: Apex fundamentals, declarative-vs-programmatic decision making, data modeling, UI, testing, debugging, deployment, and final revision.

## How to use these notes
- Read one section at a time and run each example in a Scratch Org or Developer Org.
- For each topic, write at least one variation of the example from memory.
- Maintain your own `my-observations` section under each heading as you learn edge cases.

## Core Exam Mindset
- Prefer declarative tools when they fully satisfy the requirement.
- Use Apex for complex, reusable, bulk-safe, integration-heavy, or transaction-sensitive logic.
- Always think about security: sharing, CRUD, FLS, and input validation.
- Always think in bulk: collections, maps, grouped SOQL, grouped DML.
- Always test behavior, not just coverage.
